# Frontend Interview Challenge — Product Dashboard

Welcome! You have **~30 minutes** for the coding portion of this interview.

## Setup

```bash
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

## The Brief

You're building a **product dashboard** for an internal tool. The product data is already available in `src/data/products.ts`. Your job is to build the UI in `src/App.tsx`.

### Required (Tasks 1–3)

1. **Product Grid** — Display products in a responsive card grid. Each card should show the image, name, price, rating, category badge, and stock status.

2. **Search & Filter** — Add a search input (filters by name, real-time), a category filter, and an "In stock only" toggle.

3. **Sort** — Let users sort by price (low/high) and rating (high to low).

### Stretch (Task 4 — pick any)

If you finish early, pick one or more:

- Empty state when no products match
- Keyboard-accessible controls (a11y)
- Product detail modal/drawer on click
- URL query-param sync (filters survive refresh)
- Card enter/exit animations
- Dark mode toggle

## Ground Rules

- You **may** install any npm package.
- You **may** use AI tools (Copilot, ChatGPT, Claude, etc.).
- We care about **how** you use AI, not whether you use it.
- Please **think aloud** as you work — explain your decisions.

Good luck!
